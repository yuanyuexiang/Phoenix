Why Phoenix · 为什么需要它

单据处理的每一分钟,
都不该花在抄写和翻找上。

发票、合同、确认单源源不断地进来,却以图片和 PDF 的形态沉睡在文件夹里—— 要用的时候靠人工录入、靠翻找、靠问同事。Phoenix 把这条路彻底走通。

BEFORE / 传统方式

人工逐字段抄录

看着票面敲 Excel,录得慢、录得累,还会录错。

质量靠事后抽查

错了往往在月底对账时才被发现,回溯成本高。

找单据靠翻文件夹

"上个月那张发票在哪?"——问遍同事,搜遍网盘。

出了问题无从追溯

谁录的、谁改的、什么时候入的库,说不清。

发给专家,自动识别

规则校验 + 人工兜底

Product Form · 产品形态

两个入口,一套能力。

员工在熟悉的 WorkBuddy 里对话式办事,管理员在 Web 后台把关兜底—— 底层是同一个文档处理平台。

主入口 · PRIMARY

WorkBuddy「文档处理专家」

平台以 MCP 连接器接入 WorkBuddy,交付形态是一位随叫随到的"文档处理专家": 发文件、说需求,识别、校验、入库、查询在对话里一气呵成。

  • 识别与字段提取由 WorkBuddy 客户端多模态大模型完成,充分利用终端 AI 算力
  • OAuth 免密登录:首次使用自动跳转企业统一认证,无需配置密钥
  • 大文件走 file_url 直传,不占用对话上下文
辅助入口 · CONSOLE

Web 管理后台

面向单据管理员与审核员的控制台,企业蓝设计语言,明暗双主题。

  • 文档 —— 全量单据的结构化查询与筛选
  • 审核 —— 校验未通过单据的人工修正与入库
  • 单据类型 —— 每类单据的字段与规则一目了然
  • 服务状态 —— 各组件健康度实时监控
upload_document

归档原始文件到对象存储,登记待处理状态,返回文档 ID。

extract_fields

下发该类型单据的字段抽取清单;类型未定时返回目录供选型。

validate_document

对识别结果做 schema 预校验,入库前先把问题暴露出来。

save_database

服务端权威校验后落库,正文自动切片写入知识库。

query_document

按类型、状态、上传人、字段值精确/比较条件结构化查询。

ask_document

知识库语义问答:自然语言提问,返回原文片段与出处。

六个 MCP 工具构成对外契约;新能力(如 ask_document)以新增工具方式扩展,已接入的专家无需改动。

Capabilities · 核心能力

从"收到一张图",到"查到一个数"。

六项能力覆盖单据的完整生命周期:识别、校验、审核、存储、查询、问答。

智能识别与字段提取

WorkBuddy 多模态大模型直接"看懂"图片与 PDF,按平台下发的字段清单提取结构化数据与全文。

规则校验防线

必填、格式、枚举等规则在服务端权威执行,AI 识别结果先过关、后入库,字段真实性有兜底。

人工审核兜底

校验不通过自动进入 needs_review 队列;审核员在后台修正字段、一键入库,不让问题数据落地。

结构化存储与精确查询

字段以 JSONB 落库,支持等于、包含、大小比较等字段级筛选与聚合——异构单据一套查询体系。

知识库语义问答

正文自动切片、向量化入库(pgvector);用一句自然语言提问,返回最相关的原文片段与来源单据。

单据类型配置化

一类单据 = 一份 YAML:字段、别名、校验规则声明式定义。业务扩展不改代码、不发版本。

Workflow · 处理流程

一份单据的五步旅程

从员工发出文件,到成为可查询、可问答的数据,全程自动流转、状态清晰。

上传归档

原始文件存入对象存储,登记 uploaded 状态,原件永久可溯。

识别提取

WorkBuddy 多模态大模型按平台下发的字段清单,读出字段值与全文。

规则校验

服务端按单据类型 schema 逐项核验:必填、格式、枚举一个都不能少。

入库分流

saved · 校验通过直接入库,正文写入知识库
needs_review · 未通过转人工审核台修正

查询与问答

字段级结构化查询 + 知识库语义问答,数据随取随用。

Trust · 安全与可追溯

每一条数据,都答得上"谁、何时、为什么"。

企业数据要敢用,先要可信。Phoenix 在身份、校验、审计三个层面建立信任链。

员工级身份

MCP 端点以 OAuth 2.1 对接企业统一认证,每一次调用都识别到具体员工; 支持 off / optional / required 三档开关,平滑灰度上线。

全程留痕

上传人、审核人随文档落库(uploaded_by / reviewed_by), 关键操作写入审计日志,原始文件在对象存储中永久可溯。

三重数据防线

schema 规则校验挡住格式错误,needs_review 人工审核挡住内容疑点, 审计日志让事后追责有据——AI 提效,平台守门。

Architecture · 技术架构

轻量、现代、为集成而生。

Go 微服务 + 云原生存储,以标准 MCP 协议对外;容器化交付,一条命令拉起全套环境。
