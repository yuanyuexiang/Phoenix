## 1. 产品简介

本产品用于自动处理企业文档,包括扫描件、PDF、Word、Excel、图片等。系统通过 **OCR、AI 识别和规则校验**,将非结构化文档转换为结构化数据,并自动保存到数据库和文件服务器。

**本平台以 WorkBuddy 为主要使用入口,以「文档处理专家」为交付形态。** 用户日常在 WorkBuddy 中与**「文档处理专家」**对话,完成文档的上传、识别、提取、校验、入库与查询,无需切换系统;专家底层通过 **MCP 协议**调用本平台(连接器形态)。平台自带的 Web 管理后台为**辅助角色**,主要用于规则配置、人工审核和数据管理。

**产品价值**:减少人工录入、提高数据准确率,实现文档数字化、知识沉淀与业务自动化。

## 2. 目标用户

企业行政、财务、档案管理、合同管理、采购、工程资料等需要大量处理文档的部门。**上述用户主要通过 WorkBuddy 使用本平台**;管理员/审核员则使用 Web 管理后台进行配置与复核。

## 5. 核心功能

功能 ①~⑨ 为平台提供的文档处理能力,**默认由 WorkBuddy 通过 MCP 调用触发和串联**(见功能 ⑩);同样的能力也可在 Web 管理后台手动操作,用于配置与人工审核。

| # | 功能 | 输入 | 处理 | 输出 |
|---|------|------|------|------|
| ① | 文档上传 | 扫描件/PDF/Word/Excel/图片 | 校验格式与大小,登记任务 | 文档任务记录 |
| ② | OCR 识别 | 图片/扫描件 | PaddleOCR 提取文字 | 文本内容 |
| ③ | 文档解析 | PDF/Word/Excel | 解析版式与文本 | 结构化文本片段 |
| ④ | AI 字段提取 | 文本内容 | DeepSeek/Qwen 抽取目标字段 | 字段键值对 |
| ⑤ | 数据校验 | 提取字段 | 按规则校验(必填、格式、范围等) | 校验结果与错误项 |
| ⑥ | 人工审核 | 校验结果 | 人工复核、修正(可选) | 确认后的数据 |
| ⑦ | 数据入库 | 确认数据 | 写入数据库 | 结构化记录 |
| ⑧ | 文件归档 | 原始文档 | 存入对象存储并关联记录 | 归档文件与访问地址 |
| ⑨ | 文档查询 | 查询条件 | 按条件检索 | 文档与字段结果 |
| ⑩ | **WorkBuddy 调用(主要入口)** | 用户在 WorkBuddy 中的指令 | 经 MCP 工具驱动 ①~⑨ 全流程 | 处理结果返回 WorkBuddy |

> 各功能提取的**具体字段清单**(如合同编号、金额、日期、甲乙方等)因文档类型而异,需按客户实际单据模板确认。【待确认】

## 6. 业务流程

该流程**通常由用户在 WorkBuddy 中发起指令触发**,平台经 MCP 自动串联执行,处理结果回传 WorkBuddy;人工审核环节在 Web 管理后台完成。

```
[WorkBuddy 发起] → 上传文档 → OCR识别 → 文档解析 → AI提取字段 → 规则校验
                 → 人工审核(可选) → 写入数据库 → 文件归档 → [结果回传 WorkBuddy]
```

- **人工审核**为可选环节:校验通过且置信度达标可自动入库;校验不通过或置信度偏低则转人工。置信度阈值与是否强制人工审核由客户确认。【待确认】

## 7. 系统组成

| 组件 | 职责 |
|------|------|
| **MCP Server** | **主要使用入口**,供 WorkBuddy 调用平台全部处理能力 |
| 工作流引擎 | 编排流水线各阶段,含可选人工审核 |
| OCR 服务 | 图片/扫描件文字识别(PaddleOCR) |
| 文档解析服务 | PDF / Word / Excel 内容提取 |
| AI 服务 | 基于大模型的字段提取;**模型来源可配置**(DeepSeek/Qwen API、本地部署,或客户/WorkBuddy 环境提供的模型端点)。提取逻辑始终在平台内,以保证质量与私有化数据不外流 |
| 前端管理后台 | 辅助管理入口:规则配置、人工审核、查询等 Web 控制台 |
| 数据库 | PostgreSQL,存储结构化数据 |
| 对象存储 | MinIO,存储原始文件 |
| 缓存/队列 | Redis |

## 8. 与 WorkBuddy 集成(主要使用方式)

WorkBuddy 是本平台的**主要使用入口**。平台对外提供 **MCP Server**,WorkBuddy 通过调用以下 MCP 工具完成自动化文档处理。用户全程在 WorkBuddy 中以对话/指令操作,无需登录本平台。

### 8.1 MCP 工具清单

下方接口字段为**建议草案**,最终以双方接口评审为准。【待确认】

| 工具 | 作用 | 主要入参(草案) | 主要出参(草案) |
|------|------|------------------|------------------|
| `upload_document` | 上传文档 | 文件、文档类型 | 文档 ID、任务状态 |
| `extract_fields` | 提取字段 | 文档 ID | 字段键值对、置信度 |
| `validate_document` | 校验文档 | 文档 ID / 字段 | 校验结果、错误项 |
| `save_database` | 数据入库 | 文档 ID、确认字段 | 入库记录 ID |
| `query_document` | 查询文档 | 查询条件 | 文档与字段结果 |

### 8.2 典型使用场景

1. 用户在 WorkBuddy 中提交一份合同扫描件,并说明需要提取的信息。
2. WorkBuddy 依次调用 `upload_document → extract_fields → validate_document`,平台完成识别、提取与校验。
3. 校验通过(或经人工审核后),WorkBuddy 调用 `save_database` 完成入库,并把结果反馈给用户。
4. 用户后续可在 WorkBuddy 中直接用 `query_document` 检索已处理文档。

> 具体对话交互形式、工具调用编排由 WorkBuddy 侧实现,平台保证 MCP 工具的稳定契约。

### 8.3 交付形态:以「文档处理专家」交付

面向客户,本产品以 WorkBuddy 中的**「文档处理专家」**作为交付形态和统一使用入口,分三层:

| 层 | 形态 | 说明 |
|----|------|------|
| 用户入口 | **文档处理专家** | 客户员工直接与专家对话完成全部文档处理;专家内固化工具编排(上传→提取→校验→入库)与交互话术,流程稳定,演示与验收路径一致 |
| 技术底座 | 连接器(MCP Server) | 专家底层调用 8.1 工具清单;由我方开发与维护 |
| 可选扩展 | 技能 | 针对客户高频单据类型(如合同录入、发票入账)固化专项流程,数量与范围待确认【待确认】 |

专家/技能配置在 WorkBuddy 侧完成,由我方设计,双方共同部署。
